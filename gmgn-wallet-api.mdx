# GMGN Wallet API

## Base URL

```text
http://127.0.0.1:3002
```

## Design rules

The local wallet API is organized around the wallet page you actually use:

- `profile` for identity and funding metadata
- `stats` for the overall wallet summary block
- `recent-pnl` for the `Recent PnL` tab
- `holdings` for the `Holdings` tab
- `trades` for the `Trades` tab
- `rankings` for recovered ranking blocks
- `wallet` as the compatibility aggregate bundle

`/api/gmgn/wallet/activity` still exists, but it is now a deprecated alias for `/api/gmgn/wallet/trades`.

## Endpoints

### `GET /api/gmgn/wallet/profile`

Wallet identity, avatar, label, tags, funding source, and social metadata.

### `GET /api/gmgn/wallet/stats`

Canonical overall wallet stats endpoint.

This is the endpoint to use when you need the full top summary of the wallet:

- balance
- total value
- total volume
- realized, unrealized, and total PnL
- win rate
- token counts
- buy and sell counts
- total fees
- average holding period
- last active timestamp
- PnL bucket counts
- market-cap bucket counts when upstream provides them

Example:

```bash
curl "http://127.0.0.1:3002/api/gmgn/wallet/stats?address=2tgUbS9UMoQD6GkDZBiqKYCURnGrSb6ocYwRABrSJUvY&period=7d"
```

### `GET /api/gmgn/wallet/recent-pnl`

Backs the `Recent PnL` tab.

This payload is derived from the holdings source, but sorted for recent PnL review. It is the right endpoint when you want the wallet's token-level PnL rows, not the current holdings snapshot alone.

### `GET /api/gmgn/wallet/holdings`

Backs the `Holdings` tab.

This is the normalized holdings table with holdings-specific sort and pagination controls.

### `GET /api/gmgn/wallet/trades`

Backs the `Trades` tab.

This is the normalized public wallet trades feed. Use this instead of `/activity` in all new integrations.

### `GET /api/gmgn/wallet/rankings`

Recovered ranking blocks such as `pnl_gt_5x`.

### `GET /api/gmgn/wallet`

Compatibility aggregate response bundling:

- `profile`
- `stats`
- `recentPnl`
- `holdings`
- `trades`
- `activity` as compatibility alias
- `rankings`
- `endpointInventory`

The aggregate endpoint now accepts `tradesLimit`. `activityLimit` still works as a deprecated alias.

Example:

```bash
curl "http://127.0.0.1:3002/api/gmgn/wallet?address=2tgUbS9UMoQD6GkDZBiqKYCURnGrSb6ocYwRABrSJUvY&period=7d&holdingsLimit=50&tradesLimit=40"
```

## Wallet page mapping

| Wallet UI block | Local endpoint |
| --- | --- |
| Wallet hero + identity | `GET /api/gmgn/wallet/profile` |
| Overall summary stats | `GET /api/gmgn/wallet/stats` |
| Recent PnL tab | `GET /api/gmgn/wallet/recent-pnl` |
| Holdings tab | `GET /api/gmgn/wallet/holdings` |
| Trades tab | `GET /api/gmgn/wallet/trades` |
| Ranking block | `GET /api/gmgn/wallet/rankings` |
| Compatibility aggregate | `GET /api/gmgn/wallet` |

## Query patterns

### Shared wallet section parameters

| Param | Type | Default |
| --- | --- | --- |
| `address` | `string` | `2tgUbS9UMoQD6GkDZBiqKYCURnGrSb6ocYwRABrSJUvY` |
| `chain` | `string` | `sol` |
| `period` | `string` | `7d` |

### Tab pagination parameters

`recent-pnl` and `holdings`:

- `limit`
- `cursor`
- `orderBy`
- `direction`
- `hideSmall`
- `sellout`
- `hideAbnormal`

`trades`:

- `limit`
- `cursor`
- `period`

## Compatibility notes

- `/api/gmgn/wallet/activity` is deprecated. It returns the same shape as `/api/gmgn/wallet/trades`.
- The local UI and docs no longer use `activity` as the primary wallet tab name.
- The local terminal uses `Recent PnL`, `Holdings`, and `Trades` as the three main wallet table tabs.

## Runtime support endpoints

- `GET /health`
- `GET /openapi.json`
- `GET /docs`

Use [GMGN Wallet Endpoints](./gmgn-wallet) for the upstream endpoint inventory that these local endpoints normalize.
