# OKX Trending API

## Canonical endpoint

```text
GET /api/discover/okx/solana/trending
```

This endpoint is for one job only:

- pull Solana trending token contract addresses from OKX

It is the source behind the local Discover `Trending` tab.

## Timeframes

- `5m`
- `1h`
- `4h`
- `24h`

## Why this endpoint exists

OKX is useful for a broad trending list, but the local terminal does not use OKX as the token-detail source.

The intended flow is:

1. discover token CAs from OKX
2. open the chosen CA in GMGN token detail APIs

## Example

```bash
curl "http://127.0.0.1:3002/api/discover/okx/solana/trending?timeframes=5m,1h,4h,24h&itemLimit=80"
```

## Response shape

Top-level:

- `ok`
- `fetchedAt`
- `cachedAt`
- `cacheAgeMs`
- `request`
- `data`

Inside `data.timeframes[{timeframe}]`:

- `totalRows`
- `tokenAddresses`
- `items`

`tokenAddresses` is the important field for downstream GMGN detail calls.
