# Pump API

## Base URL

```text
http://127.0.0.1:3002
```

## Purpose

The local Pump suite is split into two jobs:

- discover live token lists directly from Pump
- enrich a selected Pump token with detail, activity, holders, replies, chat, clips, and creator data

This keeps Pump useful as both:

- a standalone realtime dashboard source
- an address-first source you can bridge into other token-detail systems later

## Endpoint map

### Home clone

Use these for the `/pump` homepage clone that mirrors the Pump.fun desktop landing page:

- trending card strip at the top
- clickable homepage tabs in the middle
- dense token table at the bottom

| Local endpoint | Upstream source | UI block |
| --- | --- | --- |
| `GET /api/pump/home/dashboard` | aggregate | one-call payload for the full Pump homepage clone |
| `GET /api/pump/home/trending-coins` | `frontend-api-v3 /coins/top-runners` | trending hero cards |
| `GET /api/pump/home/table` | tab-specific homepage source | table rows for `Movers`, `Live`, `New`, `Market cap`, `Mayhem`, `Oldest`, `Last reply`, `Last trade` |

## Design rules

The local API deliberately normalizes Pump into the same response contract used elsewhere in this workspace:

- `ok`
- `fetchedAt`
- `request`
- `data`
- `error?`

For the homepage clone, the table rows also include normalized batch-enriched fields:

- `athMarketCap`
- `stats.5m / 1h / 6h / 24h`
- `txns24h`
- `volume24h`
- `traders24h`
- `graphPoints`

## Discover notes

There are two discover families:

- clean public JSON from `frontend-api-v3`
- live route payloads parsed from Pump route HTML / RSC data

The live lists are intentionally exposed as first-class local endpoints because Pump does not provide equally clean public JSON for those sections.

## Realtime notes

- discover lists are cache-backed with short TTLs
- token detail sections are safe to poll every few seconds
- the dashboard route is the easiest way to hydrate a single realtime Pump screen

## Example flow

1. Pull the discover dashboard:

```bash
curl "http://127.0.0.1:3002/api/pump/discover/dashboard?topRunnersLimit=12&recommendedLimit=12&liveLimit=12"
```

2. Pick a mint from any list.

3. Open the realtime token dashboard:

```bash
curl "http://127.0.0.1:3002/api/pump/coin/dashboard?mint=26HijyQRPisA96er6xWLh32TSMAHLvivTnmcmu7bpump&candleInterval=1h&candlesLimit=120&tradesLimit=50&holdersLimit=25&repliesLimit=25&clipsLimit=12"
```

Use the autogenerated `Pump` API reference group for the exact schema of each endpoint.
