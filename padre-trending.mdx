# Padre Trending API

## Canonical endpoint

```text
GET /api/discover/padre/solana/tokens
```

This endpoint is the source behind the local Discover tabs:

- `Trending`
- `Dex`
- `New`
- `Users`

## Timeframes

- `1m`
- `5m`
- `30m`
- `1h`

## Crawl method

Padre does not expose these lists as simple static links.

The local crawler does this instead:

1. opens the real Padre trending page with the saved browser profile
2. clicks the requested `tab + timeframe`
3. listens for websocket `init snapshot.gazes`
4. matches the visible DOM rows to the right snapshot
5. returns the full `tokenAddress` list from that matched snapshot

That is why this endpoint can recover real CAs instead of only scraping text labels.

## Query parameters

| Param | Type | Notes |
| --- | --- | --- |
| `tab` | `trending | dex | new | users` | Required |
| `timeframe` | `1m | 5m | 30m | 1h` | Required |
| `limit` | `integer` | Preview rows to return |
| `ttlMs` | `integer` | Local cache TTL |
| `refresh` | `boolean` | Bypass cache |

## Example

```bash
curl "http://127.0.0.1:3002/api/discover/padre/solana/tokens?tab=dex&timeframe=5m&limit=30"
```

## Response shape

Top-level:

- `ok`
- `fetchedAt`
- `request`
- `status`
- `data`
- `error`

Inside `data`:

- `tab`
- `timeframe`
- `totalRows`
- `tokenAddresses`
- `items`
- `matchedSnapshot`

`matchedSnapshot` is included on purpose so you can debug which websocket request the crawler matched to the visible page selection.
