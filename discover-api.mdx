# Discover API

## Base URL

```text
http://127.0.0.1:3002
```

## Purpose

The discover layer only solves one problem:

- find token contract addresses for Solana discovery tabs

It does **not** try to replace GMGN token detail. Once you have a CA from discover, open the GMGN token endpoints for full token analysis.

## Canonical endpoints

### `GET /api/discover/solana`

Unified endpoint used by the local Discover screen.

- `tab=trending` defaults to `source=okx`
- `tab=dex|new|users` resolves to `source=padre`

Recommended query params:

| Param | Type | Notes |
| --- | --- | --- |
| `tab` | `trending | dex | new | users` | Required |
| `timeframe` | `string` | Source-specific timeframe |
| `limit` | `integer` | Preview row count for the UI |
| `source` | `auto | okx | padre` | Optional override |
| `refresh` | `boolean` | Bypass local cache |

### `GET /api/discover/okx/solana/trending`

Source-specific OKX endpoint.

- only for `Trending`
- timeframes: `5m`, `1h`, `4h`, `24h`
- returns address-first lists by timeframe

### `GET /api/discover/padre/solana/tokens`

Source-specific Padre endpoint.

- tabs: `trending`, `dex`, `new`, `users`
- timeframes: `1m`, `5m`, `30m`, `1h`
- returns full token addresses recovered from the live page selection

## Source mapping

| Discover tab | Source | Timeframes |
| --- | --- | --- |
| `Trending` | `OKX` by default | `5m`, `1h`, `4h`, `24h` |
| `Dex` | `Padre` | `1m`, `5m`, `30m`, `1h` |
| `New` | `Padre` | `1m`, `5m`, `30m`, `1h` |
| `Users` | `Padre` | `1m`, `5m`, `30m`, `1h` |

## Response model

All discover endpoints are address-first.

Top-level fields:

- `ok`
- `fetchedAt`
- `request`
- `data`
- `status` when the source is crawler-backed
- `error` on source failure

Core data fields:

- `tokenAddresses`
- `items`
- `totalRows`
- `availableTimeframes`

Each `item` contains compact preview metadata only:

- `tokenAddress`
- `symbol`
- `name`
- `marketCapUsd`
- `liquidityUsd`
- `volumeUsd`
- `totalTxns`
- `totalHolders`
- `recentVisitors`
- `priceChangePct`

## Example flow

1. Call discover to get CAs:

```bash
curl "http://127.0.0.1:3002/api/discover/solana?tab=trending&timeframe=1h&limit=60"
```

2. Pick one `tokenAddress`.

3. Open GMGN token detail for full data:

```bash
curl "http://127.0.0.1:3002/api/gmgn/token?address=<CA>&chain=sol&resolution=1m&tradesLimit=16&holdersLimit=30&candlesLimit=120"
```

Use the autogenerated `Discover`, `OKX Trending`, and `Padre Trending` API Reference groups for the exact schemas.
