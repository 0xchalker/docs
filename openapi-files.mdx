# OpenAPI Files

These files drive the `API Reference` tab in Mintlify and can also be imported into Swagger UI, Postman, Insomnia, or other tooling.

## Files

- `docs/openapi/discover.openapi.json`
- `docs/openapi/dexscreener.openapi.json`
- `docs/openapi/gmgn-token.openapi.json`
- `docs/openapi/gmgn-wallet.openapi.json`
- `docs/openapi/pump.openapi.json`
- `docs/openapi/okx-trending.openapi.json`
- `docs/openapi/padre-trending.openapi.json`

## Generation

```bash
npm run docs:openapi:generate
```

## Validation

```bash
npm run docs:openapi:check
```

## Notes

- `GMGN` also exposes a runtime Swagger page at `http://127.0.0.1:3002/docs`
- `GMGN` runtime OpenAPI JSON is available at `http://127.0.0.1:3002/openapi.json`
- the generated specs assume local default base URLs unless you override them before generation
- the discover spec documents the unified `GET /api/discover/solana` endpoint used by the local screener UI
- the wallet spec documents `profile`, `stats`, `recent-pnl`, `holdings`, `trades`, and `rankings` as first-class endpoints
- the Dexscreener spec documents latest list feeds, search, pair lookups, token lookups, and token orders
- the Pump spec documents both route-derived discover lists and coin-level detail endpoints
- `/api/gmgn/wallet/activity` stays in the spec only as a deprecated compatibility alias for `/api/gmgn/wallet/trades`
