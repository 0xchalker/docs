# GMGN Wallet Endpoints

This page documents the upstream GMGN wallet endpoint inventory recovered from a live browser session.

If you want the normalized local API used by the dashboard, start with [GMGN Wallet API](./gmgn-wallet-api).

Validated in-browser on March 7, 2026 against:

- `https://gmgn.ai/sol/address/2tgUbS9UMoQD6GkDZBiqKYCURnGrSb6ocYwRABrSJUvY`
- `https://gmgn.ai/sol/address/69z4qTgQ5DBRTJvnQzx2h8jZhNsv5UgADotEwwKUm2JS`

## Runtime Note

GMGN is behind Cloudflare, so these endpoints are most reliable from a solved browser session or a Playwright persistent Chrome profile. The URL templates themselves are reusable for other wallet addresses.

## Wallet Overview

| Endpoint | Method | Template | Notes |
| --- | --- | --- | --- |
| Wallet stat | `GET` | `/api/v1/wallet_stat/{chain}/{wallet}/{period}` | Confirmed with `period=1d`, `7d`, `30d`, `all`. Main KPI block. |
| Wallet common stat | `GET` | `/api/v1/wallet_common_stat/{chain}/{wallet}` | Profile, tag, Twitter, funding source, follow and remark counts. |
| Wallet stat simple | `GET` | `/api/v1/wallet_stat_simple/{chain}/{wallet}` | Compact array response with `chain`, `address`, `token_num`, `balance`. |
| Smart money alias | `GET` | `/defi/quotation/v1/smartmoney/{chain}/wallet/{wallet}` | Legacy alias still live. |
| Smart money alias 2 | `GET` | `/defi/quotation/v1/smartmoney/{chain}/walletNew/{wallet}` | Older alias, still live on March 7, 2026. |
| Profit stat | `GET` | `/pf/api/v1/wallet/{chain}/{wallet}/profit_stat/{period}` | Adds `risk`, fee totals, native balance, average holding period. |

### Sample keys

- `/api/v1/wallet_stat/...` returns keys like `buy_1d`, `sell_7d`, `pnl`, `realized_profit`, `unrealized_profit`, `total_profit`, `sol_balance`, `total_value`, `winrate`
- `/api/v1/wallet_common_stat/...` returns keys like `avatar`, `name`, `tag`, `twitter_username`, `fund_from_address`, `fund_amount`

## Holdings

| Endpoint | Method | Template | Notes |
| --- | --- | --- | --- |
| Holdings list | `GET` | `/pf/api/v1/wallet/{chain}/{wallet}/holdings` | Confirmed required query fields include `order_by`. Working default: `order_by=last_active_timestamp&direction=desc&hide_small=false&sellout=true&hide_abnormal=false&limit=50`. Supports `cursor`. |
| Single holding | `GET` | `/pf/api/v1/wallet/{chain}/{wallet}/holding?token_address={token}` | Token-specific detail. |
| Wallet token info | `GET` | `/pf/api/v1/wallet/{chain}/{wallet}/wallet_token_info?token_address={token}` | New token-detail endpoint. |
| Wallet token info legacy | `GET` | `/api/v1/wallet_token_info/{chain}/{wallet}/{token}` | Older token-detail endpoint still works. |
| Wallet token trade stat | `GET` | `/pf/api/v1/wallet/{chain}/{wallet}/token_trade_stat/{period}?token_address={token}` | Works once `token_address` is passed. |
| Wallet holding info | `POST` | `/api/v1/wallet_holding_info` | Body: `{ "chain": "sol", "wallets_address": ["..."] }` |
| Person wallets pnl info | `POST` | `/api/v1/person_wallets_pnl_info` | Body: `{ "chain": "sol", "wallets_address": ["..."], "period": "7d" }` |

### Confirmed holdings defaults

Frontend bundle uses this default query for the new holdings API:

```text
order_by=last_active_timestamp
direction=desc
hide_small=false
sellout=true
hide_abnormal=false
limit=50
```

## Workspace Tooling

Use the crawler script added to this workspace:

```bash
npm run gmgn:wallet:crawl -- 2tgUbS9UMoQD6GkDZBiqKYCURnGrSb6ocYwRABrSJUvY
```

It writes a JSON file containing:

- endpoint inventory
- request defaults
- summarized response shapes
- raw response payloads for the validated wallet endpoints
